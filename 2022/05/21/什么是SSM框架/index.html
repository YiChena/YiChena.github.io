什么是SSM框架？

SSM（Spring+SpringMVC+MyBatis）框架集由Spring、MyBatis两个开源框架整合而成（SpringMVC是Spring中的部分内容），常作为数据源较简单的web项目的框架。

Spring

Spring是一个轻量级的控制反转(IOC)和面向切面(AOP)的容器框架。

作用

控制反转(IOC):将原本在程序中手动创建对象的控制权，交由给Spring框架来管理。简单来说就是让我们不用通过new来创建一个对象， 而是通过配置文件配置好对象的依赖关系等，让spring通过反射来创建对象。

面向切面(AOP):将业务逻辑的各个部分进行分离，在编写业务逻辑时可以专心于核心业务，从而提高了开发效率。
事务管理:事务管理是Spring框架中最为常用的功能之一,可以帮我们管理事务相关的资源 用注解和xml配置来管理事务

原理

动态注入让一个对象的创建可以自动的生产，在运行时跟配置文件来动态的创建对象和调用对象里的方法。

SpringMVC

Spring MVC 是 Spring 提供的一个基于 MVC 设计模式的轻量级 Web 开发框架

作用

主要与客户端进行交互

原理

客户端发送请求给前端控制器（DispatcherServlet）由前端控制器找到处理器映射器(HandlerMapping)请求对应的控制器(Controller)，处理业务(Service)后返回视图，视图解析器(ViewResolver)查询找到指定的视图,并将处理结果生成 View 视图显示给客户端

MyBatis

MyBatis是一款优秀的持久层框架，它支持定制化 SQL、存储过程以及高级映射。避免了几乎所有的 JDBC 代码和手动设置参数以及获取结果集。可以使用简单的 XML 或注解来配置和映射原生信息，将接口和 Java 的 POJOs(Plain Old Java Objects,普通的 Java对象)映射成数据库中的记录。

作用

MyBatis是jdbc操作的封装，支持定制化sql和存储过程，避免了JDBC代码。实现了高级映射，避免手动设置参数和获取结果集。可以让我们简单快速地连接和操作数据库，并结果集封装为Java对象返回，MyBatis把sql配置在Mapper.xml，当我们需要更改sql时只需要在Mapper中更改配置。
